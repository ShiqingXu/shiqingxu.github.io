重装系统不慌！Hexo 博客在新电脑上的无痛迁移与同步指南

拥有一个静态博客是记录思绪与分享技术的绝佳方式，而 Hexo 以其高效与简洁备受青睐。但当更换新电脑或重装系统时，如何恢复熟悉的写作环境，并实现多设备间的无缝同步，就成了一个不大不小的挑战。本文将带你走过在新 Windows 电脑上“复活”一个已部署在 GitHub Pages 上的 Hexo 博客的全过程，从零开始，逐一击破那些你可能会遇到的“经典”难题。

核心思路：双仓库战略

首先，我们要明确一个核心策略：使用两个独立的 Git 仓库来管理你的博客。

  1. 静态部署仓库 (username.github.io)：这个仓库大家都很熟悉，它只存放 hexo deploy 后生成的最终网站文件（HTML, CSS, JS），是公开的。
  2. 博客源文件仓库 (my-hexo-blog-source)：这是我们的关键。我们创建一个全新的、私有的 GitHub 仓库，专门用来存放博客的“灵魂”——源文件，包括你的 Markdown 文章、配置文件 _config.yml、themes 文件夹等。

通过这个策略，我们可以在任何电脑上克隆源文件仓库，恢复写作环境，并将修改同步回去，真正实现“一处修改，多处同步”。

旅程开始：从环境搭建到第一个“坑”

在新电脑上，我们首先安装了必备三件套：Node.js、Git 和你喜欢的代码编辑器。然而，当我们满怀信心地打开 PowerShell，敲下 npm -v 准备验证安装时，第一个“下马威”就来了：

npm.ps1 cannot be loaded because running scripts is disabled on this system.
